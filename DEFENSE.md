### Minute 3–5 — Data wrangling
**Show:** notebook §3 (loading) и §4 (cleaning).

Что говорить и что показывать:

* **Какие проблемы в исходных данных?**
  В sуrowой выгрузке region встречается под 16 разными строками: `Yssyk-Kul`, `Ysykkul`,
  `Issyk-Kul`, `Jalal-Abat`, `Djalal-abad`, `Osh oblast (until 2013 y. including Osh city)` и т. д.
  Плюс часть категорий публикует разрезы по демографическим группам
  (`0-6 years`, `Per population of retirement age`) вместо регионов.
* **Как обнаружил?**
  `long_raw['region_raw'].value_counts()` — выводит частоты, видно что одна область
  раскидана по нескольким строкам.
* **Что сделал?**
  В `fetch_nsc_data.py::_normalize_region()` — словарь из 25 строк, который маппит все варианты
  в 10 канонических имён. Затем фильтрую только канонические регионы для аналитической панели.
* **Пропущенные значения.**
  2018 год пустой для большинства показателей (НСК начал публиковать с 2019). Для медленно
  меняющихся индикаторов (доступ к воде, электричеству) — forward-fill внутри региона.
  Для остальных оставил NaN — не выдумываю.
* **Дубликаты:** ноль по ключу `(region, year)` — проверил `duplicated()`.
* **Выбросы:** 3·IQR — Бишкек и Ош-город всегда вне IQR по доходам и пенсиям. Это **не ошибка**,
  это структурная особенность (два республиканских города), оставил и зафиксировал.

### Minute 6–9 — Applied statistics + EDA
**Show:** notebook §6 (статистика) и §7 (визуализации).

Ключевые выкладки:

* **Описательная (§6.1).** mean / median / IQR для дохода, расхода, бедности, нормы сбережения.
* **Группировка (§6.2–6.3).** Региональный mean по доходу — Бишкек выходит на 8 000+ сом/мес,
  Нарын ниже 5 500. По годам средний доход растёт ≈ 14 % в номинале с 2019 по 2023.
* **Корреляция (§6.4).** Pearson — доход и бедность ≈ –0.7, доход и саанитария ≈ +0.6.
* **ANOVA (§6.5).** F-статистика большая, p < 0.001 — H0 (одинаковые средние бедности по регионам)
  **отвергается** на любом разумном уровне значимости.

### Minute 10–12 — Dashboard + Key insights
**Show:** запускаете `streamlit run dashboard/app.py`, переходите по табам.

> Этот дашборд — не «несколько графиков рядом». Я следовал принципу из 7chapter §04.06:
> KPI сверху → сравнение → распределение → корреляция → инсайты.

Прогон по табам:
1. **Overview** — 6 KPI-карточек (доход, расход, средняя бедность, поп. бедных, самый богатый/бедный регион), bar по доходу, donut по доле бедных, heatmap regions×years.
2. **National Trends** — линия дохода и расхода, индекс реального дохода YoY, прожиточный минимум.
3. **Regional Comparison** — переключатель индикатора в сайдбаре (11 опций), линии и box-plot, корреляционная матрица.
4. **Poverty Analysis** — scatter income↔poverty с OLS, гистограмма бедности, Δ бедности 2019→2023, ANOVA-блок.
5. **Data Explorer & Insights** — три ключевых вывода + полная таблица + скачать CSV.

Все элементы интерактивности из требований PDF присутствуют: фильтры по году и облласти (multiselect), dropdown выбора индикатора, поиск по названию региона, динамические графики, выбор временного диапазона ползунком, скачивание CSV.

**Три ключевых вывода:**

1. **Структурный gap Бишкек ↔ сельские области.** Бишкек удерживает ~150 % республиканского среднего по доходам каждый год 2019–2023. Это не временный эффект — gap не сужается.
2. **Самая высокая бедность — Нарын и Баткен.** Средняя по году 38–45 % vs 6 % в Бишкеке, разрыв 6–7×. ANOVA подтверждает значимость, p < 0.001.
3. **Доход ↔ бедность: сильная отрицательная связь.** Pearson r ≈ –0.7, p < 0.001 на всём наборе (region × year). Эмпирическая основа для аргумента: рост зарплат двигает региональную карту бедности сильнее, чем социальные трансферты.

## 3 · Q&A — ready-made answers (7chapter §04.09 list)

**Q1. Почему именно этот dataset?**
Тема — социально-экономика КР, источник — Нацстатком КР через публичный JSON-API, лицензия открытая.
Размер позволяет полноценный регионально-временной анализ.

**Q2. Почему он подходит для проекта?**
9 регионов × 5 лет × 15 показателей покрывают все типы анализа из §04.05 (сравнение,
распределение, тренд, связь, сводка).

**Q3. Самые большие сложности при очистке?**
1. Разнобой в названиях регионов (16 вариантов на 9 регионов).
2. Mixed-axis категории (294, 4298 публикуют разрезы по группам, а не регионам).
3. Гэп 2018 года — индикаторы начались с 2019.

**Q4. Почему такая стратегия с пропущенными значениями?**
Для медленно меняющихся access-показателей — forward-fill по региону (это распространённая
практика, ошибка минимальная). Для денежных показателей 2018 года — оставил NaN: предпочитаю
честный пропуск выдуманному числу.

**Q5. Почему именно эти производные переменные?**
`saving_rate_pct` = (доход − расход)/доход — стандартная мера фин. устойчивости.
`income_vs_national_pct` нужен для визуализации gap'а с национальным средним.
`nominal_income_yoy_pct` дополняет реальный YoY от NSC.

**Q6. Почему именно эти графики?**
Под каждый вопрос: bar/box для сравнения, line для тренда, hist для распределения,
scatter для связи, heatmap для матрицы и pivot heatmap для двумерного среза.

**Q7. Зачем нужен дашборд, а не просто графики?**
Он связывает индикаторы интерактивно — переключение по индикатору в сайдбаре одновременно
обновляет линию, бокс, ранг и корреляцию. Это коммуникативный инструмент для лица, принимающего
решения, не «отчёт».

**Q8. Какие три самых сильных вывода?**
Сверху, в Section 2 minute 10–12. Кратко: gap Бишкека, бремя Нарына/Баткена, r=–0.7
income ↔ poverty.

**Q9. Корреляция или визуальная связь?**
Корреляция: Pearson r = –0.70, p-value < 0.001 — статистически значимо. Но это **не**
причинность. Чтобы говорить о причинности, нужны квази-экспериментальные данные.

**Q10. Какие ограничения у данных?**
См. Section 13–15. Главные: только агрегаты, короткое окно, номинальные деньги.

**Q11. Что улучшили бы при большем времени?**
1. Microdata KIHS (по заявке через IHSN-каталог).
2. Geopandas + GeoJSON областей КР для choropleth.
3. Кросс-валидация с показателями WB для тех же лет.
